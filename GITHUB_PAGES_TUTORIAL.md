# Quick GitHub Pages Tutorial for API Docs

This walkthrough publishes docs generated by pdoc from your docstrings.

This is an optional, later-stage workflow. Start with [README.md](README.md) and return here only when your project has useful docstrings and you want to publish documentation.

## 1. Generate local docs

```bash
make init
make docs
```

After this, local HTML docs are in the `docs/` directory.

## 2. Preview docs locally (optional)

From the repository root:

```bash
python -m http.server --directory docs 8000
```

Then open <http://localhost:8000>.

## 3. Publish to GitHub Pages (manual path)

This repository currently ignores `docs/` in [.gitignore](.gitignore).

If you want to publish directly from `/docs`, first remove the `docs/` ignore line from [.gitignore](.gitignore), then commit generated docs.

1. Commit and push your repository to GitHub.
2. In GitHub, open repository Settings -> Pages.
3. Under Build and deployment, choose Source: Deploy from a branch.
4. Choose branch: `main` (or `master`), folder: `/docs`.
5. Save.

GitHub will build and publish a public URL like:

`https://YOUR_USERNAME.github.io/YOUR_REPO/`

## 4. Keep docs updated

Whenever docstrings change:

```bash
make docs
```

Then commit and push updated `docs/` output if you are publishing from `/docs`.

## Alternative: Publish with a GitHub Actions Workflow

This repository includes an automated workflow at [.github/workflows/pages.yml](.github/workflows/pages.yml).

One-time setup on GitHub:

1. Push this repository (including the workflow file) to GitHub.
2. Open repository Settings -> Pages.
3. Under Build and deployment, set Source to GitHub Actions.
4. Save.

After setup, docs publish automatically on pushes to `main` or `master`.

Manual trigger option: In GitHub, open Actions -> Publish Docs -> Run workflow.

This automated approach does not require committing generated `docs/` output.
